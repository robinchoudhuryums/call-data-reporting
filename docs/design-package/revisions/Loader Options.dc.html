Loaders — pick a system

Six candidates, all pure-CSS (transform/opacity only — no JS, no layout thrash, cheap on Apps Script's webview). Each is tagged with where it fits. The idea is one shared set: a full-view loader, an inline/button loader, and a skeleton for known-shape content — replacing today's four parallel idioms. All shown light + dark; the live boards below animate.

Light
1 · Refined ring
The current spinner, polished. Universal fallback — small slots, modals, buttons.
2 · Dot pulse
Compact, calm. Great inside buttons (Sending…) and tiny inline slots.
3 · Sweep bar
Indeterminate progress. Sits at the top of a panel that's refreshing in place.
4 · Glyph pulse
Per-module identity — clock for Time Clock, clipboard for Intake, etc. Full-view enters.
5 · Equalizer
Playful echo of the metrics bars / shift histogram. For Metrics + Clock loads.
6 · Skeleton shimmer
Best for known shapes — note stacks, tables, teammate cards. Feels fastest.

A suggested system (mix & match)
Full-view enter4 · Glyph pulse with the module's own icon + a mono label — gives each tool a moment of identity, replaces the two competing full-view idioms.
Lists / tables / cards6 · Skeleton shimmer — kills the text-only "Loading…" jank in Call Notes stacks and the pop-in on the Clock teammate card & histogram.
In-place refresh3 · Sweep bar at the panel top (admin panels, coverage body) — content stays visible while it updates.
Buttons / tiny slots2 · Dot pulse or 1 · Ring — replaces ad-hoc "Sending…" text swaps with a consistent inline mark.
All six share two CSS custom props (--lo-accent, --lo-track) so they theme automatically in dark mode — same mechanism as the existing .spinner. Keep the bobbing-envelope Save & Compose moment as-is; it's a deliberate flourish.
