The set

Five loaders, each for a job

Live animations. The first three are the "interesting" ones (telephony-flavored); the last two are the workhorses. Use the right one for the wait — shape-matched skeletons for content, motion loaders for indeterminate queries.

✆
1 · Signal rings
Concentric rings radiate from a handset dot — reads as "ringing / querying the line." Best for Caller Lookup and QCD run, where you're querying and there's no shape to mimic yet.
.ds-loader--rings · indeterminate query
2 · Equalizer bars
A bouncing audio waveform — small and inline. Best for button busy states ("Refresh", "Send alerts", "Look up") and compact inline spots where a skeleton would be overkill.
.ds-loader--eq · inline / button
3 · Drawing sparkline
A trend line draws itself over a faint target band — "plotting your data." Best as the chart-area placeholder while Chart.js lazy-loads or the series computes, so the chart box never sits blank.
.ds-loader--spark · chart slot
4 · Skeleton shimmer
Your existing approach, formalized with a moving sheen and shapes that match the final layout. Use for the 3 text-only spots — Alerts, Outlier Fix, Dept Config — and the Inbound tables, so there's zero layout shift on arrival.
.ds-skeleton · content-shaped
Warming up the database…
first request in a while · usually a few seconds
5 · Staged progress
An indeterminate shimmer bar with a named stage and a calm sub-line. Use for app boot and Neon cold-starts — the wait reads as expected, not broken. Swap the label as stages complete ("Warming up…" → "Fetching calls…" → "Crunching numbers…").
.ds-loader--staged · cold-start / boot · honest, not a dead spinner
If you only do three things
  1. Add the signal-rings loader to Caller Lookup & QCD — those are the spots that currently freeze on a query, and the rings are both on-theme and reassuring.
  2. Upgrade Alerts / Outlier Fix / Dept Config / Inbound from text to shape-matched skeletons — they already have a loading container; it's a swap, not new plumbing.
  3. Drop the drawing-sparkline into every chart slot while Chart.js lazy-loads (pairs with the perceived-speed playbook) so charts never pop in over blank space.

Rule of thumb: known shape → skeleton (no spinner); indeterminate query → motion loader (rings / equalizer); long/cold → staged progress with a label. Keep one spinner duration and the reduced-motion fallback for all of them.
