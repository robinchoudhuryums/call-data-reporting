Part 4

Charts — unify & de-risk

Every chart should come from one factory so the target band, focus line, palette, and labels are identical everywhere — and so a CDN hiccup degrades gracefully instead of throwing.

De-risk checklist
  • Pin exact versions and restore SRI hashes, or self-host the three libs in the project (no CDN at runtime). The dropped datalabels SRI is a silent-failure vector.
  • Keep html2canvas-pro — the original can't parse oklch() and breaks every export path.
  • Load Chart.js lazily, only when a chart route opens (see §5).
  • Register plugins once, globally; never per-chart.
  • Wrap every new Chart() in the unavailable-state guard.
Shared conventions
  • Palette = tokens. Series colors read from CSS vars, so charts re-theme with dark mode for free.
  • Target band (92% / 5%) drawn by one plugin on every applicable chart.
  • Focus line thicker + full-opacity; context series at .5 opacity.
  • End-point label on the focus series instead of a busy legend.
  • prefers-reduced-motion → animation off; maintainAspectRatio:false.
// charts.js — one factory, one source of truth
const css = n => getComputedStyle(document.body).getPropertyValue(n).trim();
const reduce_motion = matchMedia('(prefers-reduced-motion: reduce)').matches;

// register plugins ONCE
Chart.register(ChartDataLabels);
Chart.defaults.set('plugins.datalabels', { display:false });   // opt-in per chart

// target-band plugin (92% answer-rate / 5% abandon)
const target_band = {
  id:'target_band',
  beforeDraw(c,_,o){ if(!o.from&&!o.to) return;
    const {ctx,chartArea:a,scales:{y}}=c;
    ctx.save(); ctx.fillStyle=css('--good-soft');
    ctx.fillRect(a.left, y.getPixelForValue(o.to), a.right-a.left,
                 y.getPixelForValue(o.from)-y.getPixelForValue(o.to)); ctx.restore(); }
};

function lineChart(canvas, { labels, series, focus, band }) {
  return new Chart(canvas, {
    type:'line',
    data:{ labels, datasets: series.map(s => ({
      label:s.name, data:s.data,
      borderColor: css(s.token),
      borderWidth: s.name===focus ? 2.6 : 1.5,
      borderCapStyle:'round', tension:.35, pointRadius:0,
      opacity: s.name===focus ? 1 : .55,
    })) },
    options:{
      maintainAspectRatio:false,
      animation: reduce_motion ? false : { duration:400 },
      plugins:{ legend:{ display:false }, target_band: band ?? false },
      scales:{ y:{ grid:{ color:css('--line-2') }, ticks:{ color:css('--muted') } },
               x:{ grid:{ display:false }, ticks:{ color:css('--muted') } } },
    },
    plugins:[target_band],
  });
}

// graceful fallback — never let a missing lib blank the panel
function renderChart(el, cfg){
  if (typeof Chart === 'undefined') { el.outerHTML =
    '<div class="ds-banner ds-banner--neutral">Chart unavailable — data shown in the table below.</div>';
    return; }
  try { lineChart(el, cfg); } catch(e){ console.warn('[chart]', e); /* same fallback */ }
}
Part 5

Perceived-speed playbook

Apps Script + Neon means real latency (cold starts, google.script.run round-trips). The goal isn't to make it faster everywhere — it's to make the dashboard never look frozen and land on content instantly for returning users.

1 · Stale-while-revalidate the Overviewbiggest win

Cache the last successful Overview payload in localStorage. On load, paint it immediately with a "showing cached · refreshing…" pill, fire the live fetch, then swap and stamp "updated just now". A returning manager sees real numbers in <100ms.

2 · Progressive render

Return + paint KPIs and the at-a-glance banner first (cheap), then charts. Lazy-inject Chart.js only when the first chart is about to render — most Overview glances never need it.

3 · Debounce + cancel stale responses

Rapid date-range changes fire overlapping google.script.run calls that can resolve out of order. Debounce ~250ms and tag each request; ignore any response that isn't the latest token.

4 · Prefetch on intent

On hover/focus of a report tab, warm its data in the background so the modal opens populated. Pairs with the existing Neon keep-warm trigger.

5 · Honest freshness + cold-start affordance

Your freshness pill already exists — extend it with cache/refresh states. On a known Neon cold start, show "warming up…" rather than a dead spinner, so the wait reads as expected, not broken.

6 · Keep skeletons matched to final layout

You already do this well — just make the skeleton shapes match the new KPI/card grid so there's zero layout shift when data arrives.

// stale-while-revalidate (item 1)
const KEY='ov-cache-v1';
function loadOverview(){
  const cached = JSON.parse(localStorage.getItem(KEY) || 'null');
  if (cached) { renderOverview(cached.data); setFreshness('cached', cached.at); }   // instant paint
  google.script.run
    .withSuccessHandler(data => {
      localStorage.setItem(KEY, JSON.stringify({ data, at: Date.now() }));
      renderOverview(data); setFreshness('live', Date.now());                        // swap + stamp
    })
    .withFailureHandler(() => { if(!cached) showError(); })                          // keep cached on failure
    .getCompanyOverview();
}

// debounce + token (item 3)
let token = 0;
const refresh = debounce(() => {
  const mine = ++token;
  google.script.run.withSuccessHandler(d => { if (mine === token) renderTable(d); })
    .computeSummary_(from, to);
}, 250);

Part 9

Motion

Restrained, purposeful motion. The rule: 120–400ms, ease-out, transform/opacity only, play once (never loop), and every animation must mean something — data arrived, a value changed, or you navigated. All gated on prefers-reduced-motion. The previews below loop so you can see them; in production each fires once.

/* 9 · count-up — rAF, integer steps, ~400ms, first load only */
function countUp(el, to, ms=400){ if(reduce_motion){ el.textContent=to; return; }
  const t0=performance.now(), from=0;
  (function f(t){ const k=Math.min(1,(t-t0)/ms);
    el.textContent=(from+(to-from)*(1-Math.pow(1-k,3))).toFixed(1);
    if(k<1) requestAnimationFrame(f); })(t0); }
Guardrails → animate only transform & opacity (never width/top — they thrash layout); cap chart animation to first paint so data refreshes stay instant; one-shot accents add a class then remove it on animationend; and the global reduced-motion block makes every one of these a no-op for users who ask.
