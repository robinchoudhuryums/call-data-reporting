Design deep-dive · for Robin

Department Dashboard — visual modernization

The dashboard already has good bones: a coherent cool-blue palette, a real type system (Inter Tight / Inter / IBM Plex Mono), dark mode, and genuinely thoughtful data plumbing. The opportunity is purely visual & hierarchical — it currently reads as a dense wall of mono numbers where everything competes for attention. Below, each screen is shown as it is today (faithfully recreated from your CSS) next to a proposed redesign that keeps the bones and elevates the surface.

Problem 01
Clutter
Every tile shows 5+ stats at once. Nothing is the headline.
Problem 02
Readability
10 right-aligned mono columns, no emphasis, hard to scan.
Problem 03
Flat & a bit boring
Hairlines only, no depth, no status color, no focal point.
Approach
Same palette, more air
Hierarchy, status color, soft depth, progressive disclosure.
Screen 01

Overview page

The default landing. Today the dept tiles cram a big percentage, three mono stat-rows and a sparkline into a small box. The redesign makes answer rate the hero, encodes health with a status color & bar, demotes secondary stats to a clean footer, and gives the trend chart a target band so "are we OK?" is answerable at a glance.

Screen 04 · most important

Insights report

Insights is the report that answers "what changed and who needs my attention?" — a department rollup plus a per-agent card for every agent, each comparing the selected window to the period right before it. It's the right concept, but three things blunt it today:

Finding 01
Status is hidden in a border
Improved / regressed / mixed is encoded only as a thin colored left edge — and "improved" is blue, which clashes with the green you use for "good" everywhere else.
Finding 02
No triage order
Cards render in roster order, so a manager has to scan every one to find the agent who slipped. The whole point is to surface that fast.
Finding 03
"Vs prior" is implicit
The comparison baseline drives the whole report but is easy to miss. Deltas need a clear ▲▼ system and a visible "vs prior 14 days" label.

Screens 05–09 · admin & operations

Inbound · Alerts · Outlier Fix · Caller Lookup · Dept Config

The admin tools share one trait: they're stacks of dense tables where the actionable thing (assign a queue, map an outlier, see who's abandoning) is buried in columns. These users want density — so the fix isn't to strip it, it's to lead with the action or the answer, add status color, and demote pure-reference rows. Same cool palette throughout.

05 · Inbound report

Four numeric tables stacked (by insurer / dial-in / queue / dial-in×insurer). The redesign keeps every table but adds a KPI + outcome-split hero, makes "By insurer" the primary view with volume bars and colored abandon-%, and moves the other three breakdowns into tabs so the page isn't a mile long.
