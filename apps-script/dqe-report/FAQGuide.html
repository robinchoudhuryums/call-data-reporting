DQE Guide

  • Agent Names: Employees will only appear if they were rung on the selected date.
  • Total Unique vs Total Rung: "Unique" refers to a single caller attempt, while "Rung" refers to the number of times someone was notified of an incoming call. For example, if a caller is waiting on hold for 2 minutes and disconnects, that will register as 1 "unique", but several "rings". "Total Missed" & "% Answered" are based off "Total Rung", not "Total Unique".
  • TTT & ATT: "Total Talk Time" & "Average Talk Time" (only counts talk time for inbound calls from a call queue).
  • Abandon Highlight: The missed call timestamp table (times are in PST) will "highlight" timestamps that were part of an abandoned call with a red siren emoji.
  • X2 & Z2: Set start/end dates for desired data range.
  • Employee Select: Use dropdown to focus on a single employee (useful to share an individual's numbers without including other employees).
  • Quick Set Dates: There are menu options under "DQE Tools" to quickly set start & end dates to last 30, 60, or 90 calendar days.
  • Compare Employee: Tool to pick two periods for a specific employee and compare to see changes and evaluate progression over time.
  • Alerts: Automatic alert emails are sent to department leaders if their team's Answer % dropped below specific thresholds on the previous workday.

FAQ

  • Why is my table missing information? Check that your X2 (start) and Z2 (end) date range include possible dates for the desired employee(s).
  • Why didn't I get an alert email? It only triggers if Answer % is below specified thresholds (and if B1 date matches the previous workday, i.e it is possible an alert was not sent if your department's sheet was not updated the next day).
  • Why does my agent not appear in the dropdown selector? The dropdown only lists agents who had calls in the selected date range.
  • Why do I see 'N/A' for Answer %? This means no calls were (officially) routed, so Answer % is not calculated.
  • Why are there more missed call timestamps for an employee than the official "Missed Calls" count? That is because those rings were missed, but did not meet the criteria to officially count on record (usually because it was before or after official office hours).
  • Why is there a siren next to a timestamp, but the Q's Abandoned Call count is 0? This can be for a couple of reasons, most commonly that it was technically abandoned but it occurred outside of office hours, or because the abandoned calls were manually removed from the record as exceptions after review.
  • How come the Q shows 1+ Abandoned Calls, but no timestamps are highlighted with a siren? This will happen if a call was abandoned, but did not actually ring anyone because no agents were available in that timeframe.
  • What if an agent is in multiple Qs? Currently agents' data is not separated by Q, so if they appear in multiple, the day's data will be the same on both pages (this will eventually be updated).
  • The numbers for an agent don't seem to add up? (Ex: 1+ "Total Missed" but 100% "% Answered") Currently if an agent misses a call to their direct extension, it will show under "Total Missed", but does not actually count against them in "% Answered" This discrepancy is intentional, but please notify Robin if something else appears incorrect.
  • Can I adjust thresholds? Contact Robin to adjust any thresholds for a department.
