The three pages

The dashboard has three pages, switched from the header:

  • Overview (the landing page) — a company-wide snapshot: a tile per department with answer rate, abandonment and a 30-day trend chart. Admins also see a company aggregate hero. Click a department tile to jump into it.
  • My Department — the per-agent table for one department, plus its Queue Call Data tiles and per-agent missed-call timelines.
  • Escalations — the escalation worklist for your department: filter, review each escalation's activity timeline, resolve with a note, or reopen.

Reports (Individual, Insights, QCD, etc.) open as modals over any page.

Navigating & reports

The header row of tabs opens each report. Every report modal has an ↗ Open in new tab button (top-right) for side-by-side viewing, and most carry an Export menu (Download CSV / Copy image / Email / Print).

The floating ? button (bottom-right) opens this Help from anywhere, even over a report. Admin-only tabs (Alerts, Outlier Fix, Dept Config) are hidden for managers.

Dates, presets & freshness

  • From / To default to the most recent day with data, so the table isn’t empty before today’s import runs. If you set a From date past the To date, the To date snaps to match.
  • Reports offer quick presets (Yesterday, Last month, etc.); picking custom dates clears the preset.
  • The freshness pill in the header (“Data through … · Nh ago”) turns orange if the most recent data is more than ~36h old — usually a sign the daily import hasn’t run.

Themes, dark mode & Help

  • ◐ Dark mode toggles light/dark; your choice is remembered on this device.
  • ⚙ Settings picks a light-mode background theme (Warm Paper / Cool Blue / Clinical White) and toggles the floating Help button.
  • Hovering a metric label (with a small ⓘ) shows its definition; key metrics show a richer tooltip with the 92% / 5% benchmark.

Column meanings

  • Unique vs Rung — “Unique” is one distinct caller attempt; “Rung” is how many times an agent’s phone was rung. One overflow call can ring several agents. Missed and % Answered are based on Rung.
  • Missed vs Answered — rings not picked up vs answered, inside the work window. A missed ring isn’t a lost call — another agent may have answered it.
  • TTT · ATT — Total and Average Talk Time on answered queue calls.
  • Avg Abd Wait · CSR Avg Abd Wait — average time abandoning callers waited, dept-wide and CSR-specific.

The work window is 8:30 AM–5:00 PM CST (shown as a pill on the page). Rings outside it appear in the Missed timeline but don’t count toward Missed/Answered.

Source & floaters

The Source column tags each agent’s provenance:

  • ROSTER — on this department’s roster.
  • QUEUE — a “floater” from another dept who handled your queue extensions (their home dept follows the chip). Useful for spotting cross-coverage.
  • BOTH — rostered here and matched via queue this period.

Department totals are roster-only; queue-only floaters are summed on a separate footer line so headline numbers stay stable. The Diagnostics panel lists roster agents with no data in range.

Prior-period chips

Rung / Missed / Answered each show a small delta chip comparing the selected window to the same-length window just before it. Green = a good move (more answered, fewer missed), orange = a watch (more missed, fewer answered); tiny changes (under 3 calls) stay muted. Hover a chip for the prior dates and exact numbers. Chips aren’t included in the CSV export.

Queue Call Data & missed timelines

Above the agent table, a Queue Call Data tile row shows each mapped queue’s abandoned % and violations for the latest data date. Below the table, the Missed Calls section gives an hour-of-day chart (click a bar to drill into that bucket) plus each agent’s missed-call timeline listing the timestamps of unanswered rings (abandoned ones flagged in red). It follows the page’s From/To dates — change them and Refresh for a custom range (this section is the Missed Calls report; the old standalone report was folded in here).

Download CSV

The Download CSV button (appears once data loads) exports the current table view — respecting your date range and sort order — with no server round-trip. Agent names are sanitized so spreadsheet apps can’t treat them as formulas.

Individual Report

Per-agent KPI tiles (with a “Team” average marker, share-of-dept, and a sparkline) plus three trend chart tabs (Volume / Efficiency / Duration) and a 12-month window. Pick 2+ agents for a peer-comparison view with a shared legend — click a legend chip to isolate one agent across all charts. The ALL button stacks all three charts; click a tab to return to one. Optionally compare each agent against their own prior window (None / year-ago / preceding period / custom).

Insights

A team rollup + per-agent cards + 12-month trend, with comparison modes (preceding period / same window a year ago / custom). It absorbed the retired Performance Report: the KPI tiles compare against the immediately-preceding same-length window, the share-of-answered donut is below the trend chart, and the per-agent Chart view’s Absolute toggle shows each agent’s stacked Answered + Missed volumes with % answered dots. On each agent card:

  • The colored left rail and the ▲ Improving / ▼ Slipping / ≈ Mixed pill show that agent’s trend vs the prior period (across rung, missed, answered, % answered). Hover the card and the rail glows.
  • The above team / below team chip shows their standing — answered calls vs the team average this period (the marker on the Answered bar). So a card can be “Improving” yet “below team”: better than last period, still under the average.

The bars compare % Answered / Answered / Missed against the team average. A Queue health section adds the dept’s abandonment picture. If the two compared windows differ a lot in length, raw-count insights are dropped and a note appears.

Compare Ranges

Side-by-side comparison of two arbitrary date ranges (e.g. this month vs the same month last year). When the ranges differ in length by 20%+, a banner appears and per-day figures are shown so volume isn’t compared apples-to-oranges.

Missed Calls

Lives on the My Department page, below the agent table (the standalone report was folded in there). An 18-bucket bar chart (8 AM–5 PM CST, 30-minute buckets) plus per-agent cards listing each missed ring’s timestamp; it follows the page’s From/To dates. Calls the caller actually abandoned are flagged in red with a 🚨. For callers who gave up in the queue (vs a ring another agent answered), see the abandonment metrics in QCD / Inbound.

QCD (Queue Call Data)

Per-queue call volume, answered, abandoned, abandoned %, longest/avg wait, and violations (days a queue’s abandoned % went over 5%). Lives in the Insights report’s Queue health section (the standalone QCD report was consolidated into it): per-queue rows expand into per-call-source detail and violation dates, the trend chart’s “by queue” tab shows one line per queue plus a dept total, and a short days-to-violation forecast appears when a queue is trending toward the 5% line. Everyone also gets the all-departments Daily Call Queue Report from the Overview page.

Inbound Calls admin

Every distinct inbound call in the range: KPI tiles (total / answered / abandoned / on-hold / avg wait), breakdowns by insurer, dial-in number and entry queue, and a daily trend. Insurer labels come from the insurance-number block in the roster sheet. Currently admin-only while its abandonment numbers are vetted against QCD (the two use different sources and definitions).

The 92% / 5% standards

Two company-wide benchmarks drive the color-coding everywhere:

  • 92% answer rate — the target. Values at/above read green “on track”; below read orange “watch”.
  • 5% abandonment — the ceiling. A queue over 5% on a day is a violation.
94% answered vs 6.2% abandoned
Green = at/above the 92% answer target; orange = at/over the 5% abandon ceiling. These are the tints you’ll see on KPI tiles and cells across the app.

These are the only two thresholds used for tinting — counts and durations stay neutral. Department-specific alert thresholds are separate (see Alerts).

Metric glossary

  • % Answered — answered ÷ calls that rang. Target 92%.
  • Abandoned % — queue calls where the caller hung up before being answered. 5%+ in a day = violation.
  • ATT — Average Talk Time per answered call. Longer isn’t automatically worse.
  • TTT — Total Talk Time (sum of an agent’s own talk on answered calls).
  • Violations — days a queue went over the 5% abandon line (month-to-date in totals).
  • Avg / longest wait — time callers waited before being answered or giving up.

Hovering most labels in the app shows these definitions inline.

Why ATT can differ

The My Department table shows ATT as the simple mean of each day’s stored value. The reports (Individual / Compare / Insights) use a per-call weighted average (total talk ÷ calls answered), so days with no answered calls don’t drag it down. Same range can therefore show slightly different ATT in the two places — neither is wrong; they answer different questions.

Low answer-rate alerts admin

Per-department email alerts when a day’s answer rate falls below that dept’s threshold. Preview a date, then send manually, or install a daily 8 AM trigger (weekends and configured skip-dates are skipped). The Alert Log records every outcome, and a “last 30 days” chip flags depts where the threshold may be too strict or too loose.

Manager digests admin

Scheduled email summaries per manager — daily, weekly, or monthly — in a KPI “summary” format or the richer “insights” format. Managed from the Alerts modal’s Manager Digest Subscribers section; install the triggers there.

Dept Config admin

Maps each department to its queue names (the A_Q_* identifiers from the phone system) and a few per-dept options, without a redeploy. If a dept’s QCD/Inbound view is empty or missing queues, map them here — the modal auto-discovers queue names seen in recent data and flags unmapped ones.

Outlier Fix admin

Cleans up agent-name variants. If the phone system spells an agent slightly differently (e.g. a different parenthetical), this maps the variant to the canonical roster name so their numbers consolidate. Also handles new-hire roster additions. Every change is logged.

Caller Lookup admin

Look up one phone number’s inbound call history over a date range, rendered as a timeline per call (the journey through IVR / queues / agents). The number is hashed for the query and never stored or logged.

Access & who sees what

Access is by an Email → Department mapping. Managers see only their own department; admins can switch departments and see admin-only surfaces (company aggregate, Alerts, Dept Config, Outlier Fix, Caller Lookup, Inbound). Anyone not mapped gets an access-denied page with no data.

Common questions

  • My table is empty. Check the From/To dates first. If they look right, the Diagnostics panel’s “Roster agents with no data in this range” lists who was expected but absent.
  • Numbers look stale. Check the header freshness pill — if it’s orange (36h+), the daily import likely hasn’t run. Reports also cache for up to 30 minutes.
  • More missed timestamps than the Missed count. Rings outside the 8:30 AM–5 PM CST work window show in the timeline but don’t count toward Missed.
  • An agent shows in two departments. That’s expected — agents can be on multiple rosters and appear under each.
  • Inbound vs QCD abandonment differ. They use different sources and definitions; Inbound is being vetted, which is why it’s admin-only for now.

Daily trigger

The daily trigger runs at 8 AM and checks the previous day's answer rate per active dept. Weekends (Sat / Sun) are skipped automatically.

Neon keep-warm

Pings Neon every few minutes during a weekday business-hours window so the free-tier instance stays warm and DQE reads (when DQE_READ_SOURCE=neon) don't pay a cold-start penalty. Window defaults to 7 AM–1 PM Central; tune via the NEON_KEEPWARM_START_HOUR / NEON_KEEPWARM_END_HOUR Script Properties (watch the estimated monthly compute-hours below to stay under your Neon plan).

Report cache warming

Pre-warms the Overview blob + each dept's My Department default-range summary once a day (after the morning ingest) so the first manager to open the dashboard gets a cache hit instead of a cold aggregation. Fires at CACHE_WARM_HOUR Script Property (default 9 AM Central). "Warm now" runs it on demand.
